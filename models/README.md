# models/

Compiled BPU model binaries (`.hbm`) the harness and examples consume. The code
references them ONLY by the repo-relative path `models/<name>.hbm` (no absolute
paths), env-overridable per task (`BCDL_MODELS`, `BCDL_OCR_DET`, …).

**The `.hbm` files are NOT committed** — 29 models totalling ~700 MB (incl. a
128 MB embedding model and a 116 MB depth model), too large for git
(`.gitignore` excludes `*.hbm`). Populate this dir on the board with:

```bash
scripts/fetch_models.sh        # copies from the local source locations
```

`fetch_models.sh` is the single place those source paths live (the
offline-converted models, `rdk_model_zoo`, `/opt/hobot`, and one public
download); set them in `scripts/local.env`. The build workflow excludes `*.hbm`
from sync so these survive an `rsync --delete`.

**What each file is, where it came from, which build to take, and under what
licence: [`docs/MODELS.md`](../docs/MODELS.md).** Read its *which build to take*
section before swapping any of these out — several have sibling builds that
differ only in calibration or in where the graph was cut, and the wrong one
compiles cleanly and decodes to garbage. To verify a fetch: empty this
directory, re-run the script, and run the test suite.

Measured throughput per model is in
[`benchmarks/RESULTS.md`](../benchmarks/RESULTS.md), generated by
`scripts/board_bench.py` on a clean board — not hand-written.
